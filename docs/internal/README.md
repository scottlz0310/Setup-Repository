## 🔐 アクセス制御

これらのドキュメントは以下の理由で内部限定としています：

1. **セキュリティ情報**: 攻撃者に有利な情報を含む可能性
2. **内部運用**: 外部ユーザーには不要な詳細情報
3. **例外事項**: 一般的なベストプラクティスからの逸脱理由

### 更新タイミング
- **ポリシー変更時**: 必須

1. **変更提案**: Issue作成
2. **レビュー**: メンテナー承認
3. **更新実施**: PR経由
4. **通知**: チーム内共有
